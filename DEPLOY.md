# Инструкция по развертыванию на Railway

## Быстрый старт

1. **Загрузите проект на GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin <your-repo-url>
   git push -u origin main
   ```

2. **Подключите к Railway**
   - Зайдите на [railway.app](https://railway.app)
   - Нажмите "New Project"
   - Выберите "Deploy from GitHub repo"
   - Выберите ваш репозиторий
   - Railway автоматически обнаружит настройки из `railway.json`

3. **Настройте домен**
   - В настройках проекта откройте вкладку "Settings"
   - В разделе "Networking" добавьте публичный домен
   - Ваш сайт будет доступен по адресу `<your-project>.up.railway.app`

## Что происходит при деплое

Railway автоматически выполнит:
```bash
npm install && npm run build  # Сборка проекта
node server.js                # Запуск сервера на порту 8080
```

## Переменные окружения

Не требуются! Все работает из коробки.

Опционально вы можете указать:
- `PORT` — порт сервера (по умолчанию 8080)

## Проверка работы

После деплоя:
1. Откройте публичный URL
2. Нажмите "Войти" → "Регистрация"
3. Создайте аккаунт
4. Добавьте ссылки и настройте профиль
5. Ваша страница будет доступна по адресу: `your-domain/@username`

## Настройка собственного домена

1. В Railway перейдите в Settings → Networking
2. Нажмите "Custom Domain"
3. Добавьте ваш домен
4. Настройте DNS записи согласно инструкциям Railway

## Troubleshooting

**Проблема:** Приложение не запускается  
**Решение:** Проверьте логи в Railway Dashboard

**Проблема:** Порт 8080 не отвечает  
**Решение:** Railway автоматически пробрасывает порт, убедитесь что в настройках указан правильный порт

**Проблема:** Потерялись данные после перезапуска  
**Решение:** Данные хранятся в localStorage браузера. Для production рекомендуется добавить backend с БД.
